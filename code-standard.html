Code standard

Posted on 五 03 十二月 2010 in Review

金额的使用

资金计算与处理使用Money类，禁止使用浮点数直接处理，避免精度问题，应该将其转换为以分为单位的整型。

字符串

比较

最好使用StringUtils.equal，避免出现空指针。例如：

a.equal(b);

当a为空，此时会抛出空指针异常。

枚举类

枚举类对比

  • 最好使用StringUtils.equal，避免出现空指针异常。
  • 当然，最好是能转换成相应的枚举。否则容易出现由于取错值的字段导致始终不等。 例如：
StringUtil.equals(EnableStatusEnum.BLOCK_STATUS.name(),enabledStatusStr)

注意，由于EnableStatusEnum.BLOCK_STATUS.name()为BLOCK_STATUS，所以始终不可能相等。

异常

异常处理

  • 捕捉到的异常，不允许不做任何处理。至少要记录日志，或者重新抛出。
  • 最好层的业务使用者，必须处理异常，将其转换成用户可以理解的内容，不允许直接抛出给用户。

异常信息打印

  • 异常信息在日志中的打印推荐打印e，而不是e.getMessage(),因为当为运行时异常时，getMessage返回为null。

集合

判空

建议使用工具类CollectionUtils.isEmpty()，来完成。工具类，必须判断是否为null，大小是否为空。

幂等

所有请求中，除查询操作，其他所有的操作都要做幂等控制。

多次获取变化的量

当在一个页面获取一个变量，当跳转到另一个页面，或者进入后台需要再次使用这个值时，建议复用该变化值，而不是重复获取，以避免再次获取时，数据值不一致。
复用的方式： - 加签后，通过前端传到后台。 - 通过系统后台存储的形式。
